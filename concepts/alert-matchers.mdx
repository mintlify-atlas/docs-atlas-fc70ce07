An **alert matcher** is a platform-specific integration that polls a security platform and determines whether an expected alert was raised for a given detonation.

## The `AlertGeneratedMatcher` interface

Every alert matcher implements the following interface:

```go
// pkg/threatest/matchers/alert_matcher.go

// AlertGeneratedMatcher is an interface that every integration should implement to verify whether an expected
// security alert was created
type AlertGeneratedMatcher interface {
    // HasExpectedAlert verifies on a third-party service whether an alert was properly generated for the given detonation UUID
    HasExpectedAlert(uuid string) (bool, error)

    // String returns the textual, user-friendly representation of the matcher
    String() string

    // Cleanup closes the generated alerts of a given detonation on a third-party service
    Cleanup(uuid string) error
}
```

### Method responsibilities

| Method | Called by | Purpose |
|---|---|---|
| `HasExpectedAlert(uuid string)` | Polling loop in `runner.go` | Query the platform; return `true` if the expected alert exists and contains the detonation UUID |
| `String()` | Logging and error messages | Human-readable description of what alert is expected, e.g. `Datadog security signal 'My Rule'` |
| `Cleanup(uuid string)` | `CleanupScenario` in `runner.go` | Archive or close any alerts generated by this detonation so the platform stays clean |

<Note>
  `Cleanup` is called via `defer` in the test runner, so it runs regardless of whether the scenario passed, failed, or timed out.
</Note>

## Supported matchers

### Datadog Security Signals

The only built-in alert matcher queries the **Datadog Security Monitoring** (Cloud SIEM / Cloud Workload Security) signals API.

**Construction**

```go
// pkg/threatest/matchers/datadog/types.go
func DatadogSecuritySignal(name string) *DatadogAlertGeneratedAssertionBuilder
```

Optionally chain `.WithSeverity()` to also assert on signal severity:

```go
func (m *DatadogAlertGeneratedAssertionBuilder) WithSeverity(severity string) *DatadogAlertGeneratedAssertionBuilder
```

**Examples**

```go
// Assert only on rule name
Expect(DatadogSecuritySignal("Network utility accessed cloud metadata service"))

// Assert on rule name and severity
Expect(DatadogSecuritySignal("AWS Console login without MFA").WithSeverity("medium"))
```

**Authentication**

The matcher reads credentials from environment variables at construction time:

| Variable | Description |
|---|---|
| `DD_API_KEY` | Datadog API key |
| `DD_APP_KEY` | Datadog Application key |
| `DD_SITE` | Datadog site (defaults to `datadoghq.com`) |

**How `HasExpectedAlert` works**

1. Builds a Datadog signals search query filtering for open signals that match the rule name (and optionally the severity):
   ```go
   const QueryOpenSignalsByAlertNameAndSeverity = `@workflow.triage.state:open @workflow.rule.name:"%s" %s`
   const QuerySeverity = `status:%s`
   ```
2. Fetches up to 1,000 signals from the last hour via the `SearchSecurityMonitoringSignals` API.
3. For each signal, checks whether the raw signal attributes contain the detonation UUID string:
   ```go
   func (m *DatadogAlertGeneratedAssertion) signalMatchesExecution(
       signal datadogV2.SecurityMonitoringSignal,
       uid string,
   ) bool {
       buf, _ := json.Marshal(signal.Attributes.Custom)
       rawSignal := string(buf)
       return strings.Contains(rawSignal, uid)
   }
   ```
4. Returns `true` as soon as one matching signal is found.

**How `Cleanup` works**

After a scenario completes, the matcher fetches all currently open signals and archives any that contain the detonation UUID. Signals are closed via the `PATCH /api/v1/security_analytics/signals/{id}/state` endpoint with `archiveReason: testing_or_maintenance`.

<Tip>
  Archiving signals automatically keeps your Datadog Security Monitoring queue clean and prevents detection fatigue from test noise.
</Tip>

## Multiple assertions per scenario

A single scenario can register more than one matcher using repeated `.Expect()` calls:

```go
threatest.Scenario("brute force login").
    WhenDetonating(StratusRedTeamTechnique("aws.initial-access.console-login-without-mfa")).
    Expect(DatadogSecuritySignal("AWS Console login without MFA").WithSeverity("medium")).
    Expect(DatadogSecuritySignal("Repeated failed login attempts"))
```

The runner tracks all assertions in a channel-based queue and only marks the scenario as passed when every matcher has resolved:

```go
// pkg/threatest/runner.go
remainingAssertions := make(chan matchers.AlertGeneratedMatcher, len(scenario.Assertions))
for i := range scenario.Assertions {
    remainingAssertions <- scenario.Assertions[i]
}
```
