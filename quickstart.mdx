Threatest has two modes: a **CLI** for running YAML-defined scenarios and a **Go library** for writing detection tests as Go test functions. Choose the path that fits your workflow.

<Tabs>
  <Tab title="CLI">

## CLI quickstart

<Steps>
  <Step title="Install Threatest">
    Install the CLI using Homebrew:

    ```bash
    brew tap datadog/threatest https://github.com/datadog/threatest
    brew install datadog/threatest/threatest
    ```

    Or download a binary from the [GitHub releases page](https://github.com/DataDog/threatest/releases).
  </Step>

  <Step title="Set your Datadog credentials">
    Export your Datadog API and application keys:

    ```bash
    export DD_API_KEY=your_api_key_here
    export DD_APP_KEY=your_app_key_here
    # Optional: set the Datadog site (defaults to datadoghq.com)
    export DD_SITE=datadoghq.com
    ```

    <Note>
      You need a Datadog API key and an application key with `security_monitoring_signals_read` and `security_monitoring_signals_write` permissions.
    </Note>
  </Step>

  <Step title="Write a scenario file">
    Create a file named `scenarios.threatest.yaml`. Here's an example that detonates an SSH command and checks for a Datadog Cloud Workload Security signal:

    ```yaml scenarios.threatest.yaml
    scenarios:
      - name: curl metadata service
        detonate:
          remoteDetonator:
            commands: ["curl http://169.254.169.254 --connect-timeout 1"]
        expectations:
          - timeout: 1m
            datadogSecuritySignal:
              name: "Network utility accessed cloud metadata service"
              severity: medium
    ```
  </Step>

  <Step title="Lint the scenario file">
    Validate the file before running it:

    ```bash
    threatest lint scenarios.threatest.yaml
    # All 1 scenarios are syntaxically valid
    ```
  </Step>

  <Step title="Run the scenario">
    Execute the scenario against an SSH-accessible host:

    ```bash
    threatest run scenarios.threatest.yaml \
      --ssh-host test-box \
      --ssh-username vagrant
    ```

    To save results to a JSON file for CI/CD:

    ```bash
    threatest run scenarios.threatest.yaml \
      --ssh-host test-box \
      --ssh-username vagrant \
      --output results.json
    ```
  </Step>
</Steps>

## Go library quickstart

<Steps>
  <Step title="Add the dependency">
    In your Go module:

    ```bash
    go get github.com/datadog/threatest
    ```
  </Step>

  <Step title="Set your Datadog credentials">
    Export your Datadog API and application keys:

    ```bash
    export DD_API_KEY=your_api_key_here
    export DD_APP_KEY=your_app_key_here
    ```
  </Step>

  <Step title="Write a test">
    Create a test file. This example tests a Cloud Workload Security signal triggered over SSH:

    ```go cws_test.go
    package detection_test

    import (
        . "github.com/datadog/threatest/pkg/threatest"
        . "github.com/datadog/threatest/pkg/threatest/detonators"
        . "github.com/datadog/threatest/pkg/threatest/matchers/datadog"
        "github.com/stretchr/testify/require"
        "testing"
        "time"
    )

    func TestCWSAlert(t *testing.T) {
        ssh, err := NewSSHCommandExecutor("test-box", "", "")
        if err != nil {
            t.Fatal(err)
        }

        runner := Threatest()

        runner.Scenario("curl to metadata service").
            WhenDetonating(NewCommandDetonator(ssh, "curl http://169.254.169.254 --connect-timeout 5")).
            Expect(DatadogSecuritySignal("EC2 Instance Metadata Service Accessed via Network Utility")).
            WithTimeout(1 * time.Minute)

        require.Nil(t, runner.Run())
    }
    ```
  </Step>

  <Step title="Run the test">
    ```bash
    go test ./... -v -run TestCWSAlert
    ```

    You should see output indicating the scenario ran and whether the expected signal was found within the timeout.
  </Step>
</Steps>
